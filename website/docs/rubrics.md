# Rubrics

Rubrics define the scoring dimensions for each interview type.

## Score scale

| Score | Label |
|---|---|
| 1 | Weak — significant gaps |
| 2 | Below expectations — improvement needed |
| 3 | Meets expectations — solid but unremarkable |
| 4 | Strong — clear competence |
| 5 | Exceptional — above bar, memorable |

## Hire signal

Every scorecard ends with one of:

- **Strong Hire**
- **Hire**
- **Lean Hire**
- **Lean No Hire**
- **No Hire**

## Scoring dimensions

| Dimension | What it measures |
|---|---|
| Clarity | Is the answer easy to follow? |
| Correctness | Is the technical content accurate? |
| Depth | Does it go beyond the surface? |
| Tradeoffs | Are alternatives considered? |
| Communication | Is it structured and well-delivered? |
| Seniority signal | Does it match the expected level? |
| Role fit | Does it fit this specific track? |
| Specificity | Does it use real, concrete examples? |
| Improvement plan | Is a growth path identified? |

## Available rubrics

| Rubric | Used for |
|---|---|
| `behavioral-rubric` | Behavioral and recruiter screen |
| `coding-rubric` | Coding, debugging, code review |
| `system-design-rubric` | System design |
| `frontend-architecture-rubric` | Frontend architecture, Angular, React |
| `recruiter-screen-rubric` | Recruiter screens |
| `senior-engineer-rubric` | Senior and staff-level tracks |
| `ai-engineer-rubric` | AI engineering modes |
| `communication-rubric` | Candidate questions mode |
| `project-deep-dive-rubric` | Project deep-dives |

## Feedback style

All feedback is:

- **Honest** — real weaknesses are named directly
- **Specific** — pointed at exact behaviors
- **Actionable** — every weakness has a path to improvement
- **Supportive** — firm but never discouraging
