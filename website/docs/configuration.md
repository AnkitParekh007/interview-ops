# Configuration

InterviewOps uses two configuration mechanisms:

1. **`.env`** — for secrets and provider selection
2. **`config/interviewops.yml`** — for interview preferences

## Environment variables

Copy `.env.example` to `.env` and fill in only what you need:

```bash
cp .env.example .env
```

| Variable | Required | Default | Description |
|---|---|---|---|
| `INTERVIEWOPS_PROVIDER` | No | `mock` | Provider: `mock`, `openai`, `anthropic`, or `gemini` |
| `OPENAI_API_KEY` | Only for OpenAI | — | OpenAI API key |
| `OPENAI_MODEL` | No | `gpt-4.1-mini` | OpenAI model name |
| `ANTHROPIC_API_KEY` | Only for Anthropic | — | Anthropic API key |
| `ANTHROPIC_MODEL` | No | `claude-3-5-sonnet-latest` | Anthropic model name |
| `GEMINI_API_KEY` | Only for Gemini | — | Gemini API key |
| `GEMINI_MODEL` | No | `gemini-2.0-flash` | Gemini model name |
| `INTERVIEWOPS_CONFIG` | No | `config/interviewops.yml` | Config file path |
| `INTERVIEWOPS_OUTPUT_DIR` | No | `output` | Output directory |

:::tip
Start with `INTERVIEWOPS_PROVIDER=mock`. It works with zero API keys and is
perfect for trying the tool, demos, and CI.
:::

:::caution
Never commit your `.env` file. It is gitignored by default.
:::

## YAML config

`config/interviewops.yml` is generated by `npm run init`.
Only `config/interviewops.example.yml` is committed to the repository.

Key settings:

```yaml
project:
  name: "My Interview Prep"
  default_track: "senior-frontend"
  default_mode: "behavioral"
  output_dir: "output/sessions"

model:
  provider: "mock"
  temperature: 0.4
  max_tokens: 6000

interview:
  duration_minutes: 45
  difficulty: "senior"
  include_scorecard: true
  include_improved_answers: true
  include_study_plan: true

ethics:
  practice_only: true
  block_live_cheating_features: true
  include_ethics_notice: true
  no_hidden_overlay: true
  no_real_time_answer_injection: true
```

:::danger Ethics config
The `ethics` section defaults are `true` and should not be changed.
These settings are enforced in code and validated by `npm run verify`.
:::
