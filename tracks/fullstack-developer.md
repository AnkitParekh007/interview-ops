# Full-Stack Developer Track

## Who This Is For
Engineers comfortable across the full stack — frontend UI, backend APIs, databases, and deployment. Targeting roles at startups, scale-ups, or teams that value T-shaped engineers who can ship end-to-end features.

## Expected Skills
- Frontend — React or Angular, TypeScript, CSS, responsive design
- Backend — Node.js, Python, or Go; REST and/or GraphQL APIs
- Databases — SQL (PostgreSQL, MySQL) and/or NoSQL (MongoDB, DynamoDB)
- Authentication — JWT, OAuth2, session management, role-based access
- Deployment — Docker, CI/CD pipelines, cloud basics (AWS, GCP, or Vercel)
- Testing — unit, integration, e2e across the stack
- API design — REST principles, pagination, error handling, versioning
- Security basics — OWASP top 10, input validation, parameterized queries
- Developer tooling — git, linting, formatting, Makefile/scripts
- Observability — logging, basic monitoring, error tracking (Sentry)

## Interview Focus Areas
1. System design — end-to-end feature design, data modeling, API design
2. Database decisions — SQL vs NoSQL, indexing, query optimization
3. API design — RESTful patterns, error handling, versioning
4. Authentication and authorization — how you implement and secure user accounts
5. Frontend/backend contract — how you design and evolve APIs with UI in mind
6. Deployment and CI/CD — how you ship and monitor code in production
7. Security mindset — input validation, SQL injection, XSS, CSRF
8. Trade-off articulation — when to use which tool, framework, or database
9. Testing strategy — across frontend and backend, integration points
10. Code quality — readable, maintainable, well-structured code across layers

## Common Interview Rounds
- **Recruiter Screen** (30 min): Stack breadth, team fit, role scope
- **Technical Screen** (45 min): Full-stack Q&A, breadth check
- **System Design** (60 min): Design a full-stack feature from scratch
- **Coding Round** (60–90 min): Full-stack problem or backend + frontend task
- **Behavioral Round** (45 min): Ownership, debugging in production, teamwork

## Common Weak Spots
- Frontend-heavy without production backend experience
- No SQL proficiency — can't write a JOIN or explain an index
- No understanding of auth flows beyond "I use a library"
- Deployment is "I push to GitHub and it deploys" with no deeper knowledge
- No observability or logging strategy
- Testing only the happy path
- Security awareness is purely theoretical

## Seniority Expectations
- Senior: Architectural leadership across layers, production incident experience, mentoring
- Mid: End-to-end delivery without significant hand-holding
- Junior: Able to build features in an existing full-stack codebase with guidance

## Example Question Categories
- "Walk me through how you'd build a user authentication system from scratch."
- "How do you design a REST API for a social feed with pagination?"
- "What's the difference between SQL and NoSQL? When would you pick each?"
- "How does your CI/CD pipeline work at your current company?"
- "Tell me about a production bug you diagnosed and fixed."
- "How do you handle database migrations safely in production?"
- "What security vulnerabilities do you always think about when building APIs?"

## Recommended Modes
- system-design
- behavioral
- coding
- project-deep-dive
- debugging

## Evaluation Criteria
- Genuine depth across the full stack (not just frontend with backend knowledge)
- System design clarity
- Security and reliability thinking
- Database modeling and query skills
- CI/CD and deployment confidence
- Production operations experience
