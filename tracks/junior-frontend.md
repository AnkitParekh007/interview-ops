# Junior Frontend Developer Track

## Who This Is For
Early-career developers (0–2 years professional experience) targeting their first or second frontend role. This track emphasizes fundamentals, learning mindset, and growth potential over deep expertise.

## Expected Skills
- HTML/CSS — semantic markup, box model, flexbox, grid, responsive design
- JavaScript fundamentals — variables, functions, arrays, objects, async/await, fetch
- React or Angular basics — components, props/inputs, state, event handling
- Version control — git basics, branching, pull requests, code review participation
- Browser DevTools — inspecting elements, debugging JS, network tab
- Debugging mindset — reading error messages, console.log strategy, systematic investigation
- Package managers — npm basics, understanding package.json
- Basic testing — writing simple unit tests with Jest or Jasmine
- Communication — asking good questions, giving status updates, receiving feedback

## Interview Focus Areas
1. JavaScript fundamentals — closures, `this`, async patterns, promises
2. Framework basics — components, lifecycle, event handling in React or Angular
3. CSS layout — flex, grid, common patterns
4. Problem solving — approach and process, not just the answer
5. Debugging — walk through how you'd debug a broken page
6. Learning mindset — how you pick up new things, handle being stuck
7. Code quality — readable code, naming conventions, basic refactoring
8. Collaboration — git workflow, PR review, pair programming
9. Version control — commit messages, branching, resolving conflicts
10. Career trajectory — what you've built, what you want to learn

## Common Interview Rounds
- **Recruiter Screen** (20–30 min): Portfolio walkthrough, motivation, availability
- **Take-Home** (2–4 hours): Build a small feature or component
- **Technical Screen** (45 min): JS/CSS questions, talk through take-home
- **Coding Live** (45 min): Simple problem, focus on process not correctness
- **Culture Fit** (30 min): Team values, learning culture, growth

## Common Weak Spots
- Jumping to code without clarifying requirements
- Difficulty explaining what their code does (wrote it but can't articulate it)
- No structured debugging process
- Git history that tells no story (mass commits with "fixes")
- No testing at all in take-home projects
- Underestimating communication as a core skill

## Seniority Expectations
At junior level, interviewers are primarily evaluating:
- Potential and trajectory, not perfect answers
- Ability to receive feedback gracefully
- Communication when stuck ("I don't know, but here's how I'd figure it out")
- Genuine curiosity and initiative
- Clean, readable code over clever code

## Example Question Categories
- "What is the difference between `let`, `const`, and `var`?"
- "Walk me through what happens when you type a URL in a browser."
- "How does React/Angular know when to re-render a component?"
- "Tell me about a project you're most proud of. What would you do differently?"
- "You see an error in the console you've never seen before. Walk me through your debugging process."
- "How do you center a div? What are a few different ways?"
- "What's a closure? Can you write a simple example?"

## Recommended Modes
- behavioral
- coding
- take-home-review
- recruiter-screen
- debugging

## Evaluation Criteria
- Foundational JavaScript and CSS knowledge
- Framework familiarity and learning pace
- Problem-solving approach and communication
- Debugging methodology
- Growth mindset and coachability
- Code readability and basic quality habits
