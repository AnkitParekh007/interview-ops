# Release Notes — v0.1.0

**Release date**: 2026-05-13  
**Type**: Initial release (MVP)

---

## Summary

InterviewOps v0.1.0 is the first public release — a local-first, open-source AI interview practice CLI for developers.

Practice coding, system design, behavioral, frontend architecture, Angular, React, AI-assisted engineering, and more — using your own model provider, on your own machine, with no hosted backend and no hidden server.

Works out of the box with the built-in **mock provider** (no API key required).

---

## What's Included

### CLI Commands

| Command | Description |
|---------|-------------|
| `npm run init` | Initialize workspace |
| `npm run doctor` | Validate setup |
| `npm run providers` | List providers and key status |
| `npm run tracks` | List all 9 interview tracks |
| `npm run modes` | List all 13 interview modes |
| `npm run start` | Start a mock interview session |
| `npm run simulate` | Alias for start |
| `npm run plan` | Generate a personalized prep plan |
| `npm run answer` | Evaluate a written answer |
| `npm run verify` | Validate session output |
| `npm run examples` | Copy example input files |
| `npm run demo` | Full end-to-end demo |

### Interview Tracks (9)

`junior-frontend` · `senior-frontend` · `angular-developer` · `react-developer` · `fullstack-developer` · `ai-frontend-engineer` · `ai-agentic-engineer` · `devrel-engineer` · `engineering-manager`

### Interview Modes (13)

`recruiter-screen` · `behavioral` · `coding` · `system-design` · `frontend-architecture` · `angular` · `react` · `project-deep-dive` · `debugging` · `code-review` · `ai-assisted-engineering` · `take-home-review` · `candidate-questions`

### Rubrics (9)

`behavioral-rubric` · `coding-rubric` · `system-design-rubric` · `frontend-architecture-rubric` · `recruiter-screen-rubric` · `senior-engineer-rubric` · `ai-engineer-rubric` · `communication-rubric` · `project-deep-dive-rubric`

### AI Providers (4)

| Provider | API Key Required |
|----------|-----------------|
| `mock` | No |
| `openai` | Yes |
| `anthropic` | Yes |
| `gemini` | Yes |

### Session Outputs

Every session writes 8 files: `session.md`, `questions.md`, `scorecard.md`, `feedback.md`, `improved-answers.md`, `study-plan.md`, `ethics-notice.md`, `metadata.json`

### Ethics

- Ethics notice in every session (cannot be disabled)
- `npm run verify` checks ethics compliance
- No live interview assistance features
- No hidden overlay or answer injection

## Known Limitations

- Interactive Q&A mode (ask one question, wait for answer, repeat) is not yet implemented — sessions generate a full packet in one call
- No local web UI for session review yet
- No progress tracking across sessions

These are on the [roadmap](../../ROADMAP.md).
